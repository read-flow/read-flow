* Overview

Read-Flow is a local-first document archive organiser focused on e-books and
PDF files. It consists of four Rust libraries plus a JavaScript web frontend:

- =read-flow-core= — async SQLite backend, REST API, file scanning, tag management
- =provider= — generic async dependency-injection and observable-cache primitives
- =epub= — EPUB3 parsing, content modelling, and rendering protocol
- =read-flow= (=cosmic/=) — COSMIC desktop GUI
- =pwa/= — SvelteKit / TypeScript Progressive Web App

* Provider Library

** DONE Core trait
The =Provider<T>= trait exposes a single async =provide() -> Result<T, Error>=
method. Implementations are composable via combinator methods.

** DONE Combinators
- =map(f)= — synchronous value transformation
- =and_then(f)= — chained async computation that may fail
- =or_fallback(p)= — try self, fall back to =p= on error
- =cache()= — wrap in an in-memory =Cache<T>=
- =expiring_item_cache()= — cache that evicts stale entries

** DONE Observable cache
=ObservableCache= wraps a =Provider<T>= and a transformation =Fn(T) → R=.
It stores the result in a =RwLock<Option<R>>= and broadcasts an
=Invalidated= event when =set_expired()= is called. Concurrent calls use
double-check locking so the underlying provider is invoked at most once per
generation.

** DONE Sync mirror
All async traits and combinators have a parallel synchronous variant for use
in non-async contexts.

** DONE Fallback provider
=FallbackProvider= tries its primary provider; on error it tries its
secondary. The last error is returned if both fail.

* EPUB Library

** DONE EPUB3 container parsing
Parse =META-INF/container.xml= to locate the OPF package document.

** DONE Package document parsing
Parse the =.opf= file: metadata (title, author, language), manifest
(resources), and spine (reading order).

** DONE Navigation
Parse both EPUB3 =nav.xhtml= and EPUB2 =toc.ncx= into a hierarchical
=NavEntry= tree (label, href, children).

** DONE Resource resolution
Resolve href strings relative to the OPF base path and extract raw bytes
from the EPUB ZIP archive.

** DONE HTML-to-content-block parser
Convert HTML5 chapter documents into a structured =ContentBlock= tree:
- Block elements: paragraphs, headings (h1–h6), blockquotes, lists
  (ordered/unordered/definition), tables, figures, preformatted code,
  horizontal rules
- Inline elements: styled spans (bold, italic, underline, strikethrough,
  colour, font-size, font-family), links, images, SVG
- Transparent containers (=div=, =section=, =article=, =aside=) promote
  their children unless given a meaningful CSS role

** DONE CSS parsing
Parse inline =style== attributes, =<style>= blocks, and linked stylesheets.
Resolve class and tag selectors; later rules win. Supports: =color=,
=background-color=, =font-size= (px/em/%),
=font-weight= (bold/numeric), =font-style=, =text-decoration=,
=text-align= (left/right/center; justify falls back to left),
=font-family= (monospace recognised), margin, padding.

** DONE Image handling
Resolve relative image hrefs; fall back to alt-text when the resource is
missing. Recognise raster images and inline SVG; pass SVG =viewBox= aspect
ratio to the renderer.

** DONE Table rendering
Parse =<table>= with optional =<thead>=, =<tbody>=, =<tr>=, =<th>=, =<td>=
into a =ContentBlock::Table=. Cell children are themselves content blocks.

** DONE Renderer abstraction
=RenderCommand= / =RenderEvent= protocol decouples content from rendering
backend. Implementations can target COSMIC (=cosmic_text=), MuPDF, or other
surfaces.

* Core Library (read-flow-core)

** Database

*** DONE Schema
SQLite with WAL mode, foreign keys enabled, pool size up to 5. Migrations
live in =core/migrations/= and run automatically at startup via
=sqlx::migrate!=.

Tables: =files= (id, path, type, size, fingerprint, status), =file_tags=
(file_id, tag), =remotes= (base_url, order, passphrase, user_id),
=reading_progress= (fingerprint, progress JSON, last_updated).

*** DONE Reading status
Each file tracks one of three statuses: =Unread=, =Reading=, =Read=.

*** DONE Reading progress
Arbitrary JSON blob keyed by file fingerprint. Stores page/chapter
position, bookmarks, etc. Upserted on viewer close; loaded on open.

*** DONE Tag management
Files have zero or more string tags stored as rows in =file_tags=. Tags are
add-only on rescan (existing tags are not removed when a file is re-scanned).

*** TODO Book collections
A =collection= entity (DB migration) grouping the files found under one
directory tree, populated at scan time by mapping directory trees to
collections. Browsing/filtering by collection needs matching UI on both
COSMIC and PWA.

** File Scanning

*** DONE Three-stage async pipeline
1. *Traversal* — recursively walk configured directories; filter by
   extension; skip hidden files, hidden directories, and SCM roots
   (=.git=, =.hg=). Emit (path, auto-tags).
2. *Fingerprinting* — SHA256-hash each file concurrently (up to configured
   worker count). Emit =ScannedFile=.
3. *DB writer* — upsert file rows (=INSERT OR IGNORE=); batch-apply tags;
   emit =ScanProgress= events (=FileDiscovered=, =FileProcessed=,
   =FileError=, =Completed=).

*** DONE Dry-run mode
Traversal and fingerprinting run, but no DB writes are made.

*** DONE Auto-tagging
Directory-level glob patterns map to tag sets. Any file whose path matches
a pattern receives the associated tags on scan.

*** DONE Supported types
PDF, EPUB, MOBI, FictionBook (FB2), CBZ, CBT, XPS/OpenXPS, DOCX, XLSX,
PPTX, HWPX, and a catch-all =Other= variant.

*** DONE Documents inside archives
The scanner detects archive files (=.zip=, =.tar=, =.tar.gz=/=.tgz=,
=.tar.bz2=/=.tbz2=, =.tar.xz=/=.txz=, =.tar.zst=/=.tar.zstd=/=.tzst=) and
indexes matching document files found inside them. Archive members are
fingerprinted (SHA256 of the member bytes), get metadata and cover
extraction via a temp file, and are stored in =files= with the synthetic
unique path ={archive_path}::{inner_path}= plus dedicated =archive_path= /
=archive_inner_path= columns. The REST download endpoint and the local
open action extract the member on demand. Tar-based archives (unlike zip)
have no random access, so when a scan needs multiple members from one tar
archive, =tar_single_pass= spools all wanted members to a temp dir in a
single decompression sweep instead of re-decompressing per member.
Nested archives (archives inside archives) are not supported yet; the
=::= path encoding and the =core/src/scan/archive.rs= module are designed
to allow recursing later.

*** TODO Nested archives
Recurse into =::=-separated inner segments so an archive found inside
another archive is scanned and indexed too, extracting each level to
memory/temp before descending. Tracked inline as a TODO in
=core/src/scan/archive.rs=.

*** TODO Full-text content search
Index document body text (EPUB content blocks, PDF text layer already
extracted by MuPDF) and extend search beyond filename/path/tags to match
document content. Current search (=documents.search=, DONE) only covers
filename, path, and tags.

*** TODO Media category expansion
The scanner only knows document types (=DocumentType= in
=core/src/scan/mod.rs=). Introduce top-level media categories — documents
(=.doc=/=.docx=/=.pdf=/=.epub=/=.mobi=/etc.), images (=.jpg=/=.png=/=.webp=/etc.),
videos (=.mpeg=/=.mov=/=.mp4=/etc.), audio (=.ogg=/=.mp3=/=.wav=/etc.) — each
with its own extension set and, eventually, its own viewer.

*** TODO Audit log for scan changes
The async scanner already logs file updates (size/fingerprint changed)
and new tag insertions at =INFO= level with all the before/after data
needed. Persist that as an =audit_log= table (migration required) plus a
COSMIC page to inspect what changed between scans and when.

*** TODO Calibre library import
Recognize Calibre library directories during scan and support importing
metadata from a Calibre database (=metadata.db=) instead of relying only
on filesystem scanning.

** REST API

*** DONE Authentication
HTTP Basic Auth with per-user PBKDF2-hashed passwords stored in config.

*** DONE File endpoints
- =GET /files= — list all files with tags
- =GET /files/<guid>= — single file with tags
- =GET /files/tags= — all tags in use
- =PUT /files= — update file (status, metadata)
- =POST /files= — upload multipart file; auto-scan and import
- =DELETE /files/<guid>= — remove from DB and filesystem
- =GET /files/<guid>/cover= — cover image
- =POST /files/<guid>/document= — resolve/create the document a file belongs to

*** DONE Tag endpoints
- =POST /files/<id>/tags= — add tags
- =DELETE /files/<id>/tags= — remove tags

*** DONE Download endpoint
=GET /files/<guid>/download-as/<filename>= — stream file bytes.

*** DONE Reading-progress endpoints
- =GET /reading-state/<fingerprint>=
- =PUT /reading-state=

*** DONE Document endpoints
Documents group one or more files (sources) that represent the same work
(e.g. an EPUB and a PDF of the same book, or duplicate scans).
- =GET /documents= — list documents with their sources
- =GET /documents/<guid>= — single document detail
- =PUT /documents/<guid>/metadata= — edit title/authors/description/etc.
- =GET /documents/<guid>/cover= — document cover (from the selected source,
  or overridden — see =select_cover=)
- =POST /documents/merge= — merge two or more duplicate documents into one

*** DONE Source management endpoints
- =DELETE= a single source/file of a document (=sources.delete=)
- Send a document to another configured client — download-to-local or
  upload (=sources.send_to_client=)
- Sync a document to all configured sources, COSMIC-only (=sources.sync_to_all=)
- Choose which source's cover represents the document (=documents.select_cover=)

*** DONE Admin endpoints (owner role required)
- =POST /scan= — trigger a library scan; returns a =ScanSummary=
- =POST /maintenance/check-missing= — detect and purge DB rows whose
  underlying file no longer exists on disk
- =GET/PUT/DELETE /scan-directories= — CRUD scan directory configuration
- =GET/PUT /settings= — server settings (db path, file types, dry-run, private mode)
- =GET/POST /users=, =PUT/DELETE /users/<user_id>= — CRUD authorised server users

*** DONE Online library (OPDS) endpoints
- =GET /online-library/search= — search configured OPDS catalogs
- =POST /online-library/import= — download and import a book

*** DONE Remote status
Each remote server exposes =GET /status=; the PWA and COSMIC clients probe
a configured remote's =/status= to show reachability before use.

*** DONE TLS
The server optionally terminates HTTPS directly (rustls) when a
=[server.tls]= cert/key pair is configured; plain HTTP otherwise. TLS
settings are operator-local (COSMIC + config file only, not exposed over
the REST =ServerSettingsDto=).

*** TODO Mutual TLS (client certificates)
One-way TLS is done (server speaks HTTPS, Preferences has a server-TLS
section including self-signed cert generation). Remaining: client
certificate verification on the server, client certificate presentation
in the =client= module, and extending sources + Preferences TLS settings
to configure client certificates.

* read-flow-widgets Library

A dedicated crate of reusable COSMIC-compatible widgets not provided by
libcosmic. Its first and primary widget is a collapsible tree widget used as
the application nav sidebar.

** Tree Widget

*** DONE Node / leaf model
The tree is composed of two kinds of items:
- *Node* — has one or more child items. Nodes render a collapsible section.
- *Leaf* — has no children. Leaves render as a single interactive element.

The distinction is structural: a node and a leaf are different types at the
widget level, not a runtime boolean flag.

*** DONE Collapsible nodes
Each node has a right-aligned icon-button that toggles its children between
collapsed (hidden) and expanded (visible). The icon indicates the current
state (e.g. =go-next-symbolic= when collapsed, =go-down-symbolic= when
expanded). Clicking the icon-button does not trigger the node's primary
action.

*** DONE Visual differentiation of nested elements
Child items are visually distinct from their parent via indentation or a
similar visual cue. The exact treatment follows the COSMIC design language.

*** DONE Composable leaf buttons
Leaf nodes accept any button-like element as content — they are not
restricted to a fixed text+icon template. This makes the widget suitable for
different content types (document entries, chapter headings, page
thumbnails, etc.).

*** DONE Message-type mapping
The widget is generic over its message type and supports a =.map()=
combinator (mirroring =iced::Element::map=) so that inner message types can
be lifted to an outer type at the call site. This allows pages to define
their own sub-tree message types independently of the top-level
=App::Message=.

*** DONE Programmatic collapse/expand
The collapsed/expanded state of any node can be set programmatically, not
only via user interaction. This allows the application or an individual page
to control tree state in response to events (e.g. collapse all nodes when
switching pages).

*** DONE Page sub-tree ownership
Each =Page= implementation owns and manages its own sub-tree of nav items.
The tree widget is assembled at the =App= level by composing each page's
sub-tree, so pages remain independent of each other's items.

* COSMIC Desktop GUI (read-flow)

** Document List

*** DONE Display
Paginated table with columns: filename, type icon, size, reading status,
tags. Reads from the aggregated local + remote document set.

*** DONE Search
Toggle between fuzzy and regex search modes. Search targets filename and
path. Ctrl+F focuses the search input.

*** DONE Sorting
Sort by: filename, type, path, size, fingerprint. Toggle ascending /
descending. Ctrl+M persists the sort preference.

*** DONE Tag filtering
Include-filter (file must have all listed tags) and exclude-filter (file
must have none). Status filter (Unread / Reading / Read). Filters compose
with AND semantics.

*** DONE Batch operations
Multi-select files → bulk add/remove tags or bulk delete.

*** DONE Preferences persistence
Sort column, sort direction, and search mode are persisted across sessions.

*** DONE Merge duplicates
Select two or more documents identified as duplicates and merge them into
one, combining their sources and tags.

** Document Details (context pane)

*** DONE Metadata display
Type icon, filename, path, size, reading status, tags.

*** DONE Metadata editing
Edit title, authors, description, and other metadata fields; persists via
=PUT /documents/<guid>/metadata=.

*** DONE Status editor
Radio-button status selector (Unread / Reading / Read) updates the DB
immediately.

*** DONE Tag editor
Inline tag editor: add tags by typing, remove individual tags. Changes
persist immediately.

*** DONE Open / delete actions
Open with the native viewer (PDF / EPUB) or the system default application.
Delete removes from DB and filesystem.

*** DONE Source management
For documents with multiple sources (e.g. an EPUB and a PDF of the same
book): delete an individual source, choose which source's cover
represents the document, send a source to another configured client, or
sync a document to all configured sources.

*** TODO Configurable duplication location
Basic push/pull between local and remote already works ("send to
missing" pushes to a remote lacking the file; pulls to local when only a
remote copy exists), but always to/from a fixed location. Add: a
per-file destination picker instead of the global =download_folder=; a
way to specify the target folder/path on the remote for uploads; and a
way to explicitly copy a file to a remote that already has it (forced
duplicate).

*** DONE Keyboard shortcuts display
Relevant keyboard shortcuts for the active viewer are shown in the context
pane.

** PDF Viewer (MuPDF)

*** DONE Rendering
MuPDF-backed rasterisation of PDF and reflowable documents. Renders one or

*** DONE Navigation
Previous/next page buttons; jump-to-page input; keyboard arrow keys.

*** DONE Chapter navigation

*** DONE Image viewer
Tapping an image opens a fullscreen image-viewer page.

*** DONE Error display
When the EPUB cannot be opened, a centred error icon and localised message

*** DONE Viewer backend choice
COSMIC-only setting to choose which EPUB viewer backend opens a document:

** Custom Nav Bar Widget
:PROPERTIES:
:SECTION-NOTE: Not yet implemented. Uses the read-flow-widgets tree widget.
:END:

The libcosmic =nav_bar= widget (backed by =SegmentedButton=) does not support
per-item trailing buttons or a collapsible-section API. The sidebar is
replaced by an instance of the =read-flow-widgets= tree widget (see above).

*** DONE Replace libcosmic nav bar with the tree widget
Override =Application::nav_bar()= to return the tree widget from
=read-flow-widgets= instead of =crate::widget::nav_bar(...)=.

*** DONE Styled to match COSMIC nav bar
The tree widget must visually match the existing libcosmic nav bar: colours,
corner radius, item height, spacing, and padding pulled from =theme.cosmic()=.
Active, hovered, and inactive button states match =SegmentedButton::NavBar=
style.

*** DONE Scrollable
The full item list is wrapped in a scrollable container (=Length::Fill=
height), identical to current nav bar behaviour.

*** DONE Body and chevron are independent click targets
Each node row has two distinct interactive zones:
1. *Body* (icon + label) — navigates to the page.
2. *Chevron* (right-aligned icon-button) — toggles collapsed/expanded state
   without navigating.

Leaf items have no chevron.

*** DONE Pages manage their own sub-trees
Each =Page= implementation is responsible for building and updating its own
sub-tree of nav items. The current =nav_sub_entry_mappings= / =NavSubEntriesChanged=
mechanism in =App= is replaced: =App= assembles the final tree by composing
each active page's sub-tree, and pages handle their own collapse/expand state
and item updates internally.

*** DONE No layout shift on toggle
Collapsing or expanding a node must not cause the parent item to jump or the
scroll position to change unexpectedly.

** DONE Theme editor

** DONE Desktop per-app theme overrides
Preferences → Appearance lets users theme the COSMIC app (dark/light,
accent, density, roundness, frosted glass, interface font, advanced
background colours) without changing global COSMIC settings. A light
and a dark colour profile (accent, background, container background)
are both configured and stored at once, in =[ui.theme.light]= and
=[ui.theme.dark]= of read-flow.toml; the app switches between them
live to match whichever mode the system is currently in, rather than
being pinned to one. Density, roundness, frosted glass, and interface
font are shared across both variants. Applies live (font family via
the in-process CosmicTk global) except font size (restart). Frosted
glass is Linux-only (=cfg(target_os = "linux")=; needs compositor
blur) — controls hidden and mapping skipped elsewhere.
Feature id =app.theme_overrides=.
