#+TITLE: Provider Crate 📦
#+DESCRIPTION: A flexible Provider pattern for lazy value computation with caching and expiration support.

* Current Implementation

** Core Components

| Component                 | Description                                   |
|---------------------------+-----------------------------------------------|
| ~Provider<T>~ trait       | Core abstraction for value provisioning       |
| ~Value<T>~                | Simple cloneable value wrapper                |
| ~ExpiringValue<T>~        | Value with atomic expiration flag             |
| ~Cache<T, P>~             | Lazy caching wrapper with manual invalidation |
| ~ExpiringItemCache<T, P>~ | Auto-invalidating cache based on item expiry  |
| ~MappingProvider~         | Functional transformation of provider output  |
| ~Expiring~ trait          | Expiration status interface                   |

** Modules

- *sync* - Uses ~std::sync::RwLock~ for synchronous contexts
- *async* - Uses ~tokio::sync::RwLock~ with ~trait-variant~ for async

* Future Provider Ideas 💡

** 🔄 Composition & Control Flow

| Provider                   | Description                                                                                               |
|----------------------------+-----------------------------------------------------------------------------------------------------------|
| ~FallbackProvider<P1, P2>~ | Tries primary provider first, falls back to secondary on error. Useful for graceful degradation           |
| ~ChainProvider<Vec<P>>~    | Tries multiple providers in sequence until one succeeds                                                   |
| ~RetryProvider<P>~         | Wraps provider with configurable retry logic (attempts, backoff strategy) for transient failures          |
| ~TimeoutProvider<P>~       | Adds timeout to async providers - critical for network operations in UI apps                              |
| ~ZipProvider<P1, P2>~      | Combines two providers into a tuple ~(T1, T2)~ - useful when you need data from multiple sources together |

** ⏱️ Time-Based Caching

| Provider                     | Description                                                                                    |
|------------------------------+------------------------------------------------------------------------------------------------|
| ~TtlCache<T, P>~             | Time-to-live cache - expires after a duration. Perfect for API responses, thumbnails, metadata |
| ~RefreshingCache<T, P>~      | Background refresh before expiration - keeps data fresh without blocking UI                    |
| ~StaleWhileRevalidate<T, P>~ | Returns cached value immediately while fetching fresh data in background - great UX pattern    |

** 📁 File & I/O Providers

| Provider               | Description                                                                           |
|------------------------+---------------------------------------------------------------------------------------|
| ~FileProvider~         | Reads file content, handles errors gracefully                                         |
| ~WatchingFileProvider~ | Uses inotify/notify to auto-invalidate cache when file changes - perfect for config   |
| ~CommandProvider~      | Executes shell command and provides output - useful for integrating with system tools |
| ~DirectoryProvider~    | Lists directory contents with optional filtering                                      |

** 🌐 Network Providers

| Provider                  | Description                                               |
|---------------------------+-----------------------------------------------------------|
| ~HttpProvider~            | Fetches data from HTTP endpoints                          |
| ~ThrottledProvider<P>~    | Rate-limits calls to underlying provider - prevents abuse |
| ~OfflineFirstProvider<P>~ | Returns cached value when offline, syncs when online      |

** 🖼️ Cosmic/Desktop-Specific

| Provider                  | Description                                                                      |
|---------------------------+----------------------------------------------------------------------------------|
| ~CosmicConfigProvider<T>~ | Integrates with ~cosmic-config~ for app settings with automatic change detection |
| ~ThumbnailProvider~       | Generates/caches thumbnails for files (integrates with freedesktop spec)         |
| ~IconProvider~            | Provides icons from icon themes based on file type/name                          |
| ~MimeTypeProvider~        | Determines MIME type for files                                                   |
| ~XdgProvider~             | Provides XDG directory paths (config, cache, data dirs)                          |
| ~MetadataProvider~        | Extracts metadata (EXIF, PDF info, audio tags)                                   |

** 🔔 Reactive/Observable

| Provider                   | Description                                                                   |
|----------------------------+-------------------------------------------------------------------------------|
| ~ObservableProvider<T, P>~ | Wraps provider with change notification - subscribers get notified on changes |
| ~StreamProvider<T>~        | Produces a stream of values instead of single value - for real-time updates   |
| ~PollingProvider<P>~       | Periodically polls underlying provider and notifies on changes                |

** 🛠️ Utility Providers

| Provider                  | Description                                                          |
|---------------------------+----------------------------------------------------------------------|
| ~LazyProvider<P>~         | Defers provider construction until first use                         |
| ~OnceProvider<P>~         | Guarantees provider is called exactly once (like ~OnceCell~ pattern) |
| ~FilterMapProvider<P, F>~ | Filters and maps in one step for collection providers                |
| ~FlatMapProvider<P, F>~   | Maps each item to a provider and flattens results                    |
| ~OptionProvider<P>~       | Gracefully handles ~None~ cases, converts ~Option<T>~ errors         |

* Priority Implementations for Read Flow 🎯

Based on the read-flow application context, these providers would have the most impact:

1. *TtlCache* - For document metadata, thumbnails with time-based expiration
2. *WatchingFileProvider* - For config files and document directories
3. *FallbackProvider* - For graceful handling when documents are unavailable
4. *ThumbnailProvider* - For document previews in the list view
5. *CosmicConfigProvider* - For settings integration with the COSMIC ecosystem
6. *RetryProvider* - If network/cloud storage support is added
7. *ObservableProvider* - To update UI when underlying data changes

* Usage Example

#+BEGIN_SRC rust
use provider::sync::{Provider, Value, Cache};
use provider::r#async::Provider as AsyncProvider;

// Simple value provider
let value = Value::from("Hello World!");
assert_eq!(value.provide().unwrap(), "Hello World!");

// Cached provider with manual invalidation
let cached = some_expensive_provider.cache();
let _ = cached.provide(); // Computes value
let _ = cached.provide(); // Returns cached value
cached.set_expired();     // Invalidate cache
let _ = cached.provide(); // Recomputes value

// Mapping provider (transform output)
let doubled = counter.map(|x| x * 2);

// Chaining combinators
let provider = expensive_computation
    .cache()
    .map(|data| data.process());
#+END_SRC
