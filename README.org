* Overview

Read Flow is a cross-platform document management system designed to help organize and manage files across multiple machines. The project provides multiple interfaces for interacting with your document archive:

- A *REST API server* for programmatic access
- A *Cosmic desktop application* for Linux desktop environments
- A *Single Page Application (SPA)* web interface with fuzzy search capabilities
- A *Command-line interface* for scanning and tagging operations

* Architecture

The project is structured as a Rust workspace with three main crates and a separate web application:

** Core Components

*** =core= (Core Library & Server)
The main library and server implementation providing:
- Database management using sqlx (async) with SQLite
- REST API server using Rocket framework
- File scanning and fingerprinting
- Tag management and automatic tagging
- Client API for remote server communication
- Settings and configuration management

*Key Modules:*
- =api/= - File data source and API definitions
- =client/= - Client for communicating with remote servers
- =db/= - Database models, schema, and connection pooling
- =scan/= - File system scanning and directory traversal
- =server/= - REST API endpoints and server implementation
- =tag/= - Tag management and automatic tagging logic
- =settings/= - Configuration parsing and settings management

*Binary:*
- =read-flow-cli= - Main CLI entry point with commands:
  - =scan <path>= - Scan directories for files
  - =apply-tags= - Apply automatic tagging rules
  - =serve= - Start the REST API server
  - =extract-scan-directories= - List all scan directories

*** =cosmic= (Desktop Application)
A native Linux desktop application built with libcosmic (COSMIC desktop environment toolkit):
- Multi-window support with COSMIC desktop integration
- Internationalization support (English, French, Dutch)
- Single-instance application architecture
- Async/tokio-based event handling
- Accessibility features

*Key Modules:*
- =app/= - Main application model and state management
- =component/= - Reusable UI components
- =page/= - Application pages/views
- =forms/= - Form handling and validation
- =state/= - Application state management
- =i18n/= - Internationalization resources
- =client/= - Integration with read flow backend
- =document_provider/= - Document data provisioning

*Features:*
- Private mode for sensitive documents
- Tag-based filtering and organization
- Native file system integration
- GPU-accelerated rendering (wgpu)
- Wayland and X11 support

*** =provider= (Dependency Injection Framework)
A lightweight dependency injection and observable pattern library:
- =sync/= - Synchronous provider implementations
- =async/= - Asynchronous provider implementations
- Observable caching with invalidation support
- Generic provider traits for dependency management

*** =pwa= (Progressive Web App)
A SvelteKit + TypeScript PWA that runs entirely in the browser:
- SvelteKit with =adapter-static= (no dedicated backend)
- Tailwind CSS v4 for styling
- Fuse.js for client-side fuzzy search
- PDF.js for PDF rendering, epub.js for EPUB rendering
- IndexedDB via Dexie for local storage (sources, reading progress, preferences)
- Connects to one or more remote =read-flow= servers as data sources

*Files:*
- =src/= - SvelteKit source (routes, lib, stores, API client)
- =vite.config.ts= - Vite + vite-plugin-pwa configuration
- =svelte.config.js= - SvelteKit adapter-static configuration

* Configuration

The application is configured via =read-flow.toml= with the following sections:

** =[database]=
Database connection settings:
- =url= - Path to SQLite database file

** =[server]=
Server configuration:
- =download_folder= - Temporary download directory
- =authorized_users= - Basic authentication credentials

** =[scan]=
Scanning behavior:
- =dry_run= - Enable/disable dry run mode
- =auto_tags= - Automatic tagging rules
- =directories= - Scan directory configurations with actions (Scan/Ignore), tags, and inheritance settings

** =[ui]=
User interface settings:
- =private_mode= - Enable/disable private mode
- =private_tags= - Tags to hide unless in private mode

* Features

** File Management
- Automatic file scanning and fingerprinting
- Duplicate detection via content-based fingerprinting
- Tag-based organization with automatic tagging rules
- Reading status tracking (Unread, Reading, Read)
- Multi-directory support with configurable inheritance

** Search and Filtering
- Fuzzy search with Fuse.js (web interface)
- Tag-based filtering
- Filename and path matching
- Support for typos and partial matches

** Multi-Platform Access
- Native desktop application for COSMIC desktop
- Web interface accessible from any browser
- REST API for custom integrations
- Command-line tools for automation

** Security
- Basic authentication for REST API
- Private mode for sensitive documents
- Tag-based content filtering

** Internationalization
- Desktop app supports English, French, and Dutch
- Fluent-based localization system

* REST API

The REST API (documented in =openapi.yaml=) provides:

- =GET /status= - Server status
- =GET /files= - List all files
- =POST /files= - Upload new file
- =GET /files/{id}= - Get file details
- =PUT /files/{id}= - Update file information
- =DELETE /files/{id}= - Delete file
- =GET /files/{id}/tags= - Get file tags
- Additional tag and remote management endpoints

* Database Schema

The application uses SQLite with sqlx (async). Migrations are located in =core/migrations/= as flat =.sql= files and run automatically at startup. They include:

- Initial database schema (files, tags, file_tags)
- Remote server support
- Remote ordering and authorization
- Reading status tracking
- User ID management

* Building and Running

** Prerequisites
- Rust toolchain (edition 2024)
- Node.js and npm (for SPA)
- SQLite3

** Build Commands

*** CLI and Server
#+BEGIN_SRC bash
# Build all workspace members
cargo build --release

# Run the server
cargo run --bin read-flow-cli -- serve

# Scan a directory
cargo run --bin read-flow-cli -- scan /path/to/directory

# Apply automatic tags
cargo run --bin read-flow-cli -- apply-tags
#+END_SRC

*** Desktop Application
#+BEGIN_SRC bash
cd cosmic
cargo build --release
cargo run
#+END_SRC

*** Web Interface (PWA)
#+BEGIN_SRC bash
cd pwa
npm install
npm run dev
#+END_SRC

* Development

** Workspace Dependencies
The workspace uses centralized dependency management. Common dependencies include:
- =sqlx= - Async database (SQLite)
- =rocket= - Web framework
- =serde= - Serialization
- =tokio= - Async runtime
- =clap= - CLI parsing
- =reqwest= - HTTP client
- =libcosmic= - COSMIC desktop toolkit

** Code Organization
- The project follows Rust's module system conventions
- Each crate has a well-defined responsibility
- The =provider= crate enables loose coupling via dependency injection
- Settings are managed through an observable cache pattern for reactive updates

** Testing
Test files and configurations can be found in:
- =core/src/= - Unit tests
- =rest-api.http= - HTTP endpoint tests
